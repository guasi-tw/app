# 使用情境

使用者 A
透過網站，告知在社群平台上 A 有哪些分身帳號

採用 A 主動登記、揭露的機制。
使用時機：當 A 的 a1 被 ban 時，A 可以快速告知 a2 也是他的帳號。

UI:
A 註冊帳號後，登記並告知平台 P1 的 id1 是他的帳號


== 認證機制：如何確定 A 的宣稱是真的？
在平台 P 上有此認證網站的帳號，認證過程當 A 宣稱 P1 的 id1 是他的的時候，需要透過私訊的互動來認證
確認方式： challenge authentication, A 先把平台的帳號加入好友，然後一連串私訊之後，平台便可以加入

== 揭露功能：
使用者 A 可以選擇要揭露多少他的帳號

== 瀏覽功能：
所有未登入的使用者都可以看到某帳號的所有帳號
查詢平台 P 的 id1, 會出現一個頁面告知此 id1 也是 id2.....

(類似 link tree, 又有分身, 又有備份的功能, 想要有台語諧音, whoami)
